区块链交易仿真系统设计与实现

这是一个面向教学、实验和毕业设计答辩的区块链交易仿真系统。 它不是一个真实公链，也不是生产级钱包，而是一个可以在浏览器中直观看到“钱包创建、交易签名、交易池排队、PoW 挖矿确认、区块上链、分叉产生、自私挖矿攻击”等区块链核心机制如何运转的可视化实验平台。 项目介绍文案 这个项目是一个基于 Vue 3 和 Node.js/Express 开发的区块链交易仿真系统，主要用于演示区块链从交易产生到区块确认的完整过程。系统内置钱包、交易、mempool、PoW 挖矿、区块链浏览、分叉处理和自私挖矿实验等功能，用户可以通过前端页面完成钱包创建、登录、发起交易、查看交易状态、手动挖矿、观察区块生成，并进一步分析区块链在分叉和攻击场景下的行为变化。 系统的核心目标是把抽象的区块链概念变成可操作、可观察、可演示的流程。普通用户不需要理解底层代码，也可以通过页面看到一笔交易如何被签名、如何进入交易池、如何被矿工打包进新区块、如何获得确认，以及链上出现分叉时主链和孤块是如何被识别的。对于毕业设计或课堂展示来说，这个项目可以作为一个完整的区块链机制演示平台。 这个项目主要做什么 它主要模拟了一条简化版区块链网络的运行过程。 用户首先可以在系统中创建钱包。钱包会生成公钥、私钥和地址，地址由公钥哈希生成，用于标识交易发送方和接收方。创建钱包后，用户可以登录系统并查看自己的余额、地址和交易信息。 然后用户可以发起交易。系统支持使用私钥对交易进行 ECDSA 数字签名，后端会校验交易签名、地址、余额、nonce、gas 等信息。验证通过的交易不会立刻上链，而是先进入 mempool，也就是待打包交易池。 接下来，用户可以触发 PoW 挖矿。矿工会从 mempool 中选择交易，按照优先级、gasPrice、nonce 等规则进行排序和打包，然后通过工作量证明计算新区块哈希。当新区块满足难度要求后，系统会把它加入区块链，并更新交易状态、钱包余额和区块高度。 除了普通交易和挖矿流程，项目还实现了区块链分叉管理。当多个区块在相同高度竞争时，系统可以展示主链、分叉链和孤块，让用户理解为什么区块链需要主链选择机制，以及非主链区块为什么会成为孤块。 项目中比较有特色的一点是自私挖矿实验。系统允许模拟自私矿工隐藏自己挖出的区块，在合适时机发布私有链，从而影响主链选择和矿工收益。用户可以观察自私矿工和诚实矿工的收益对比、私有链领先数量、分叉变化和孤块产生过程。 核心功能 项目包含以下几个核心模块： 钱包管理 支持创建系统钱包、生成地址、保存私钥、登录认证、查询余额。 交易处理 支持创建交易、数字签名、签名验证、nonce 校验、gas 参数、交易优先级和交易状态追踪。 mempool 交易池 交易提交后先进入 mempool，等待矿工打包。系统可以展示待处理交易，并按照 priority、gasPrice、nonce 等规则决定打包顺序。 PoW 挖矿机制 模拟工作量证明挖矿，生成新区块，发放矿工奖励，并将交易确认到链上。 区块链浏览 支持查看区块高度、区块哈希、前一区块哈希、区块内交易、主链区块和非主链区块。 交易生命周期可视化 可以展示交易从创建、签名、提交、进入 mempool、被打包、被确认的完整流程。 分叉与孤块展示 系统能够记录和展示分叉区块，区分主链区块和孤块，帮助理解区块链主链选择逻辑。 自私挖矿实验 支持开启/关闭自私挖矿、设置 gamma 参数、模拟私有链挖矿、发布私有链、比较自私矿工和诚实矿工收益。 实验控制功能 支持重置链状态、清空 mempool、生成测试数据、一键触发自私挖矿演示场景，方便答辩和课堂演示。 技术架构 这个项目采用前后端分离架构。 前端使用 Vue 3 + Vue Router + Vuex + Vite + Axios + Bootstrap，负责页面展示、用户交互、交易创建、钱包管理、区块链可视化和实验操作。 后端使用 Node.js + Express，负责提供钱包、交易、区块链、挖矿、分叉、自私挖矿和实验控制相关 API。 核心区块链逻辑主要包括： Wallet：钱包、公私钥和地址生成 Transaction：交易结构、签名和验证 Mempool：待打包交易池 Block：区块结构和哈希计算 Blockchain：主链、挖矿、余额、交易确认、分叉处理 SelfishMiner：自私挖矿攻击模拟 系统还使用本地 JSON 文件保存钱包数据和区块链状态，因此服务重启后仍然可以恢复之前的链状态，便于持续演示和调试。

Design Practice

Myarxiv论文预发布平台

myArXiv 是一个面向科研论文预发布、检索、投稿、审核、文件存证和链上凭证生成的论文预印本平台，整体类似一个带后台审核和区块链/IPFS 存证能力的 arXiv 风格系统。 下面这版可以直接给别人看，也可以整理进 README。 项目概述 myArXiv 是一个论文预发数据库系统，主要用于帮助科研用户在线提交论文、上传论文文件、填写论文元数据、选择学科分类、查看论文详情、检索论文内容，并支持论文版本更新、审核管理、用户权限管理、文件存储和链上存证等完整流程。 项目不仅包含普通用户使用的前台系统，还包含管理员、学科管理员、审核员使用的后台管理系统，以及基于 Spring Boot / Spring Cloud 的后端服务。整体目标是搭建一个从“论文提交”到“审核发布”再到“公开检索、长期存储、凭证生成”的完整论文预印本平台。 它解决什么问题 这个项目想解决的是学术论文在正式发表前的开放共享和可信管理问题。用户可以像使用 arXiv 一样，把自己的论文提前提交到平台中，平台通过学科分类、审核流程、文件管理和版本控制，让论文可以被有序归档、查询和展示。 同时，项目还引入了 IPFS 和区块链相关能力：论文文件可以通过文件服务进行存储，论文或文件信息可以生成哈希、凭证或链上记录，用来增强论文归档的可信度、可追溯性和防篡改能力。 主要用户角色 系统大致面向四类角色： 普通用户：注册登录后，可以浏览学科分类、搜索论文、提交论文、管理自己的已发布/未发布论文、更新论文版本、查看个人信息和权限申请状态。 论文作者：在普通用户基础上，重点使用投稿流程，上传论文文件，填写标题、摘要、作者、分类、许可证等元数据，并提交审核。 审核员：在后台查看待审核论文，对论文进行审核、通过、拒绝、暂缓等处理，也可以查看自己已审核或被拒绝的论文记录。 管理员/学科管理员：负责用户管理、审核员管理、学科分类管理、权限管理、统计信息查看，以及平台基础数据维护。 核心功能 前台用户端主要包括： 首页学科浏览：按照 Subject 和 Category 展示论文分类，用户可以进入某个分类查看新论文、近期论文或进行搜索。 用户注册与登录：支持账号登录、邮箱登录、邮箱验证码、忘记密码、重置密码等流程。 五步投稿流程：项目中投稿流程被拆成 Start、Add Files、Process、Metadata、Preview 五个步骤，覆盖论文提交的主要环节。 论文文件上传：支持论文文件保存、图片上传、文件下载、文件删除和文件信息更新。 论文元数据填写：包括论文标题、摘要、作者、学科分类、分类属性、许可证等信息。 论文列表与详情：用户可以查看论文列表、论文详情、论文文件、论文版本等信息。 论文检索：支持按分类、关键词或多条件查询论文。 个人中心：用户可以查看个人资料、已发布论文、未发布论文、权限申请等。 版本更新：已发布论文可以进行版本更新，保留论文版本历史。 NFT/链上凭证相关功能：项目中包含证书生成、文件哈希查询、mint 状态更新、链上记录提交等功能入口。 后台管理端主要包括： 论文审核管理：审核员可以查看待审核论文，执行通过、拒绝、暂缓、认领审核等操作。 已审核论文管理：查看已经处理过的论文记录。 拒绝/暂缓论文管理：管理未通过或被暂缓的论文。 审核员管理：管理员可以新增、编辑、删除学科审核员。 学科管理员管理：管理不同学科下的管理员。 普通用户管理：查看用户列表，冻结、恢复、删除用户账号。 学科与分类管理：维护 subject、category、category value 等论文分类体系。 权限申请管理：处理普通用户提交的权限申请。 角色与权限管理：支持角色、权限、用户角色、用户权限等 RBAC 相关能力。 统计信息：后台提供论文、用户、审核等平台数据概览。 技术架构 项目整体是一个前后端分离、多模块的系统。 前台用户端位于 develop 目录，使用 Vue 3、Vue Router、Vuex、Element Plus、Axios，并引入 Web3、Ethers 等区块链交互依赖。它负责普通用户看到的 myArXiv 网站，包括首页、投稿、论文列表、论文详情、搜索和用户中心。 后台管理端位于 background 目录，使用 Vue 3、Vite、TypeScript、Pinia、Element Plus、ECharts。它基于动态路由和角色菜单构建，适合管理员、审核员、学科管理员使用。 后端主业务服务位于 feature/cloud-myarxiv/myarxiv-project，使用 Spring Boot、Spring Cloud Alibaba、Nacos、OpenFeign、MyBatis-Plus、MySQL、Redis、Web3j。它负责用户、论文、投稿、审核、分类、权限、通知、统计、链上记录等核心业务。 文件服务位于 feature/cloud-myarxiv/myarxiv-document，主要负责文件上传、下载、IPFS 存储等能力，依赖 java-ipfs-http-client。 公共模块和服务接口模块包括 myarxiv-common 和 myarxiv-feignapi，用于放置通用响应结构、公共实体、Feign 接口等内容。 数据设计 数据库中包含比较完整的论文平台数据模型，例如： user 用户表、role 角色表、privilege 权限表、user_role 用户角色表、paper 论文表、submission 投稿表、paper_file 论文文件表、submission_file 投稿文件表、paper_audit 论文审核表、category 分类表、subject 学科表、license 许可证表、endorsement 背书表、file_md5 文件哈希表、permission_application 权限申请表、user_onchain_audit 链上审核记录表等。 这说明项目不是一个简单展示页面，而是围绕“论文提交、审核、发布、归档、检索、权限、存证”建立了比较完整的业务闭环。 典型业务流程 一个典型流程是： 用户注册并登录平台，进入首页浏览学科分类；选择投稿后，按照五步流程填写论文信息、上传论文文件、补充元数据并预览提交；提交后，论文进入后台审核流程；审核员在后台认领或处理论文，审核通过后论文进入公开论文库；其他用户可以按分类或关键词搜索、查看论文详情和下载文件；作者后续还可以更新论文版本；系统可以进一步生成文件哈希、证书或链上存证记录。 项目亮点 这个项目的特点在于它不是只做“论文展示”，而是把论文预发布平台的多个关键环节都串起来了： 它有前台用户系统，也有后台管理系统；有投稿流程，也有审核流程；有论文检索，也有分类体系；有用户权限，也有审核员/管理员角色；有传统数据库存储，也有 IPFS 和区块链相关扩展；有论文版本管理，也有文件哈希和证书生成能力。 换句话说，它可以被介绍为：一个集论文预发布、投稿审核、学科分类、论文检索、用户权限、文件存储和区块链可信存证于一体的科研论文开放归档平台。
